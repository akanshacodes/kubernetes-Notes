## Why Basic Editors Are Not Enough

Tools like **Vim** or **Notepad** work, but they are:

- Very basic with no syntax highlighting
- Prone to indentation errors
- Unable to warn you about mistakes in real time
- Unaware of Kubernetes-specific rules (wrong property names, list vs dictionary misuse, etc.)

As your YAML files grow longer or more complex, these limitations become a real problem.

## Recommended IDE — VS Code

**Microsoft Visual Studio Code** is the recommended IDE for this course because:

- Free and open source
- Works on all operating systems (Windows, macOS, Linux)
- Has a **free YAML extension** (by Red Hat) with built-in Kubernetes support
- Easy to set up and get started

> You are free to use any IDE of your choice, but make sure to find a Kubernetes-compatible YAML plugin for it.

## Installing VS Code

1. Go to [https://code.visualstudio.com](https://code.visualstudio.com)
2. The website will automatically detect your OS and suggest the right download.
3. Download and install the appropriate package for your system:

| OS | Package Type |
|----|-------------|
| Ubuntu / Debian | `.deb` package |
| Red Hat / Fedora | `.rpm` package |
| Windows | `.exe` installer |
| macOS | `.dmg` file |

**On Ubuntu/Debian, install via terminal:**
```bash
sudo dpkg -i code_*.deb
```

## Installing the YAML Extension

Once VS Code is open:

1. Click on the **Extensions** icon in the left sidebar (or press `Ctrl+Shift+X`)
2. Search for **YAML**
3. Look for the extension published by **Red Hat**
4. Click **Install** if it isn't already installed

> In many cases, this extension comes pre-installed with VS Code. If you see it listed as enabled, you're good to go.

**What this extension provides:**
- YAML syntax validation (structure, indentation, types)
- Kubernetes manifest validation (wrong field names, incorrect types, missing required fields)
- Auto-complete suggestions for Kubernetes properties

## Enabling Kubernetes Support in the Extension

By default, the Red Hat YAML extension only does basic YAML validation. To enable **Kubernetes-specific** support, you need to configure it.

There are **two ways** to do this:

---

### Method 1 — Via the Gear Icon (Per File)

When a YAML file is open in VS Code:

1. Click the **gear icon** that appears at the bottom-right of the editor
2. Select **Kubernetes** as the schema to validate against

This applies Kubernetes validation for that specific file session.

---

### Method 2 — Via VS Code Settings (Global, Recommended)

To apply Kubernetes validation for all your YAML files permanently:

1. Open VS Code Settings (`Ctrl+,` or `Cmd+,` on Mac)
2. Search for `yaml schemas`
3. Click **Edit in settings.json**
4. Add the following configuration:

```json
{
  "yaml.schemas": {
    "kubernetes": "*.yaml"
  }
}
```

This tells the YAML extension to use the **Kubernetes schema** for all `.yaml` files automatically.
